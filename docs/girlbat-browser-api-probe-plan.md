# Girlbat Browser API Probe Plan

This plan maps the browser API families from
[Issue 841](../issues/0841-browser-api-implementation-audit/01-cross-engine-browser-api-audit.md)
onto Girlbat. It is a roadmap, not a parity claim.

## Classification

- `smoke-covered`: Girlbat has a proof for the tested slice only. This does not
  mean full feature parity.
- `source-evidence-only`: Ladybird source suggests support, but Girlbat has no
  runtime probe yet.
- `explicitly-unsupported`: Girlbat has deterministic unsupported behavior and
  the behavior is cited.
- `unproven`: no reliable Girlbat runtime or source evidence yet.
- `out-of-scope-for-issue-884`: product, security, or UI work belongs to later
  issues.

`explicitly-unsupported` is intentionally narrow. At the time this plan was
written, the only browser-facing deterministic unsupported paths are devtools
creation and HTTP auth handling. Other absent browser APIs are classified as
`unproven` or `out-of-scope-for-issue-884` until Girlbat has a real denial,
probe, or product decision.

## Ranked Follow-Ups

1. **Girlbat local API probe harness.** Build a reusable harness around the
   existing loopback shape for console beacons, permissions queries, storage,
   popups, fullscreen/pointer-lock, and identity captures.
2. **Clipboard and color-scheme probes.** These are high-signal, local, and do
   not require OS modal UI when scoped to write/copy and `matchMedia` readback.
3. **Storage/profile isolation probes.** These validate the
   one-profile-per-process model beyond protocol lifecycle behavior.
4. **Popup/new-window and generic download probes.** These require product
   policy but are important for everyday browsing.
5. **Visible PDF and print/save probes.** Build on Experiment 33 after final
   visible Ghostboard runtime evidence.
6. **Security/product UI APIs.** Media capture, WebAuthn/passkeys, Payment
   Request, device APIs, notifications, push, native share, external protocol
   launch, and password-manager integration need dedicated product/security
   issues before support is claimed.
